AI 算力核心三要素

  • 浮點運算力 (FLOPS)：決定矩陣計算速度。大語言模型 (LLM) 推論主要評估 FP16/BF16 或 FP8 效能。
  • 記憶體容量 (VRAM)：決定模型載入與 Context Window 大小。80B 模型 (FP16) 僅權重即需約 160GB 空間。
  • 記憶體頻寬 (Bandwidth)：決定資料存取速度。LLM 推論屬「記憶體頻寬受限」，直接決定生成速率 (Tokens/s)。

※ 系統效能極限取決於最弱環節 (如：PCIe 頻寬瓶頸、容量不足導致的系統記憶體交換)。

NVIDIA H200 架構

  • 推出時程：2023 年發布 / 2024 年量產。
  • 實體形態：SXM5 模組 或 PCIe 擴充卡。
  • 硬體規格：144GB HBM3e (系統可用 141GB) / 頻寬 4.8 TB/s。
  • 基礎設施：單卡功耗 600W-700W，支援標準氣冷。
  • 軟體生態：搭配 x86 CPU，相容現有 OS 與 CUDA 環境。
  • 系統限制：泛用型核心。跨卡資料交換受限於 PCIe Gen5 (雙向 128 GB/s) 瓶頸。

NVIDIA GH200 (Grace Hopper)

  • 推出時程：2023 年發布。
  • 實體形態：Superchip (CPU + GPU 整合封裝)。
  • 硬體規格 (144GB 版)：624GB 統一記憶體 (144GB HBM3e + 480GB LPDDR5X) / GPU 頻寬 4.0 TB/s。
  • 基礎設施：模組功耗 450W-1000W，支援氣冷或液冷。
  • 軟體生態：底層搭載 ARM 架構 (Grace CPU)。
  • 系統優勢：內部 NVLink-C2C (900 GB/s) 消除 PCIe 瓶頸，專攻超大型記憶體需求。

NVIDIA GB200 (Grace Blackwell)

  • 推出時程：2024 年發布。
  • 實體形態：1 顆 Grace CPU + 2 顆 Blackwell B200 GPU。
  • 硬體規格：384GB HBM3e / 頻寬 16 TB/s / 支援 FP4 精度。
  • 基礎設施：模組功耗達 2700W，強制要求液冷 (Liquid Cooling)。
  • 系統優勢：兆級模型推論頂規，以機櫃級 (Rack-Scale) 部署為主。

通訊拓樸：NVLink 與 NVSwitch

  • NVLink (點對點直連)：
    取代 PCIe 的高速互連。晶片內 (如 GH200 NVLink-C2C) 達 900 GB/s；跨 GPU 直連頻寬遠超 PCIe Gen5，極大化降低存取延遲。
  • NVSwitch (交換矩陣)：
    網路交換晶片。串聯多節點或機櫃內數十顆 GPU，達成全互連 (All-to-All) 無阻塞通訊。

市售 AI 伺服器盤點 (2U~4U)

  • HPE DL384 Gen12 (2U)： 支援 Dual GH200 (NVL2)。在氣冷環境下提供雙倍運算節點與記憶體池。
  • HPE DL380a Gen12 (4U)： 雙路 x86，最多 8 張雙寬 PCIe GPU，高擴展性。
  • DELL R760xa (2U)： 雙路 x86，最多 4 張雙寬 PCIe GPU。
  • Supermicro ARS-111GL-NHR (1U)： 單顆 GH200。散熱壓力極大。
  • Supermicro SYS-421GE-TNRT (4U)： 雙路 x86，最多 10 張雙寬 GPU。

選型規劃：HPE DL384 Gen12 + Dual GH200 NVL2

1. 任務隔離：零編譯負擔

伺服器僅負責 LLM 推論，韌體編譯與 Agent 部署於外部主機。因此，轉換至 ARM 架構 (Grace CPU) 不會產生軟體相容性與編譯成本。

2. 排除 GB200

GB200 依賴機櫃級液冷與高電力負載，目前市場缺乏符合 4U 內且具備標準氣冷的單機方案。

3. 容量與頻寬優勢 (GH200 vs H200)

80B 模型需約 160GB VRAM。單張 H200 (141GB) 無法獨立載入；若採多卡 x86 伺服器，跨卡通訊將受限於 PCIe 延遲。

4. 最終效益

Dual GH200 NVL2 總計提供 288GB HBM3e 與 960GB LPDDR5X 記憶體池。透過 NVLink 互連規避 PCIe 瓶頸，在氣冷伺服器框架下，最大化推論吞吐量與 Context Window。
